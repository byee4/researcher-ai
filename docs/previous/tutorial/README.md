# Example: eCLIP-seq Pipeline Reproduction

**Paper:** Van Nostrand EL et al. (2016). Robust transcriptome-wide discovery of RNA-binding protein binding sites with enhanced CLIP (eCLIP). *Nature Methods*, 13, 508–514.
**PMID:** 26971820 | **DOI:** 10.1038/nmeth.3810 | **GEO:** GSE77695

This example demonstrates the researcher-ai output for a well-documented eCLIP-seq paper. All files here were generated by running the Phase 8 end-to-end notebook (`notebooks/07_end_to_end.ipynb`) with `paper_source = "26971820"`.

## What this paper does

The paper introduces eCLIP (enhanced CLIP), an improved method for mapping RNA-binding protein (RBP) binding sites transcriptome-wide. The key technical improvements over iCLIP are:

1. A paired size-matched input control to reduce false-positive peaks
2. Improved adapter ligation efficiency
3. Higher reproducibility across replicates

The computational pipeline processes raw paired-end FASTQ files through adapter trimming, genome alignment, UMI-based deduplication, peak calling, and normalization against input controls.

## How to reproduce

### 1. Set up environment

```bash
conda env create -f environment.yml
conda activate eclip_pipeline
```

### 2. Download raw data

The raw eCLIP data is available at [GEO accession GSE77695](https://www.ncbi.nlm.nih.gov/geo/query/acc.cgi?acc=GSE77695).

```bash
# Download using SRA toolkit (example for one sample)
prefetch SRX1521435
fastq-dump --split-files SRX1521435/
```

### 3. Run the pipeline

```bash
# Configure sample paths in config.yaml, then:
snakemake --snakefile Snakefile --cores 16 --use-conda
```

### 4. Reproduce figures

Open `figure_reproduction.ipynb` in Jupyter and run all cells after placing processed data in `data/`.

## Reproducibility notes

- Genome: hg19 (GRCh37) — the paper pre-dates hg38 adoption
- STAR version 2.4.0j used in the paper; versions 2.7.x also work with minor parameter differences
- CLIPper peak caller is not on Bioconda — see installation note in `environment.yml`
- The paper uses a custom UMI-deduplication script; the pipeline uses the `umi_tools dedup` equivalent
